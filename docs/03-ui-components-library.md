# Step 3: UI Components Library

## Goal
Créer une bibliothèque de composants UI réutilisables avec les patterns du design Escalya.

---

## Implementation Details

### Files to Create
- `components/ui/Badge.tsx`
- `components/ui/Card.tsx`
- `components/ui/Input.tsx`
- `components/ui/Textarea.tsx`
- `components/ui/CounterInput.tsx`
- `components/ui/StayCard.tsx`
- `components/ui/Hero.tsx`
- `components/ui/Testimonial.tsx`
- `lib/cn.ts` (utility clsx/tailwind-merge)

### What to Implement
1. **Badge Component**: Create reusable badges for statuses like "Member Exclusive" or "All-Inclusive".
2. **Card Component**: Build a generic card component with hover effects.
3. **Input Components**: Include `Input`, `Textarea`, and `CounterInput` for forms.
4. **StayCard Component**: Design luxury stay cards with hover zoom and overlays.
5. **Hero Component**: Implement a full-screen hero section with a gradient overlay.
6. **Testimonial Component**: Add a testimonial section for client reviews.
7. **Utility Function**: Add `cn.ts` for class merging using `clsx` and `tailwind-merge`.
